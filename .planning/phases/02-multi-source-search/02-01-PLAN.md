From literature-search reference (api_client.py), sci_search API:
- POST to `https://qyyqlnwqwgvzxnccnbgm.supabase.co/functions/v1/sci_search`
- Headers: `x-api-key: {token}`, `Content-Type: application/json`
- Payload: `{"query": str, "topK": int}`
- Response: list of paper dicts (each has title, abstract, authors, doi, url, year, citationCount, etc.)

From literature-search reference (api_client.py), OpenAlex API:
- GET `https://api.openalex.org/works?search={query}&per_page=N&sort=cited_by_count:desc`
- `mailto` param for polite pool (10 req/s)
- Response: `{"results": [...]}` with fields: id, doi, title, display_name, publication_year, publication_date, ids, open_access, authorships, abstract_inverted_index, etc.
- Abstract stored as inverted index: `abstract_inverted_index: {"word": [pos1, pos2]}`
- Open access info: `open_access.oa_url`

From literature-search reference (api_client.py), Semantic Scholar API:
- GET `https://api.semanticscholar.org/graph/v1/paper/search?query={query}&limit=N`
- Fields param: `paperId,title,abstract,authors,year,citationCount,url,externalIds,publicationDate,openAccessPdf`
- Response: `{"data": [...]}`
- openAccessPdf: `{"url": "...", "status": "GREEN"}`
- externalIds may contain DOI, ArXivId, PubMedId
- Rate limit: without API key 100 req/5min; with key 1 req/sec

        If `OPENALEX_EMAIL` set, add `mailto={email}` param.

        - `abstract`: OpenAlex stores abstracts as inverted index in `abstract_inverted_index`. Reconstruct with a helper method `_reconstruct_abstract(inv_index)` that builds a sorted word list: for each word -> list of positions, build position->word mapping, join by position order. If no inverted index, use `None`.
        - `authors`: from `work.get("authorships", [])` -> extract `author["author"]["display_name"]` for each
        - `doi`: from `work.get("doi", "").replace("https://doi.org/", "")` (OpenAlex returns full DOI URL)
        - `source`: `"openalex"`
        - `source_url`: from `work.get("doi")` or `work.get("id")` (OpenAlex URL)

     f. Run keyword searches in parallel via `asyncio.gather` with `return_exceptions=True`
     g. Deduplicate by `paper_id` within this adapter
   - Semaphore: 5 concurrent requests (OpenAlex is generous with polite pool)

   Helper `_reconstruct_abstract(inv_index: dict) -> Optional[str]`:
   ```python
   if not inv_index:
       return None
   positions = {}
   for word, pos_list in inv_index.items():
       for pos in pos_list:
           positions[pos] = word
   return " ".join(positions[k] for k in sorted(positions.keys()))
   ```
